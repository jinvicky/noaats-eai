# NOAATS-EAI — 기획서

> 이기종 시스템 간 데이터 연계를 웹 UI에서 설정·실행·모니터링하는 EAI 플랫폼

---

## 기획 배경

기업 환경에서 시스템 간 데이터 연계는 반복적으로 발생하지만, 매번 개발자가 직접 연동 코드를 작성해야 하는 비효율이 존재합니다.

NOAATS-EAI는 **어댑터·인터페이스·트리거를 웹 UI에서 조합**하여, 코드 변경 없이 시스템 간 데이터 흐름을 정의하고 운영할 수 있는 플랫폼입니다.

---

## 해결하려는 문제

| 기존 방식 | NOAATS-EAI |
|-----------|------------|
| 연동마다 개발자가 직접 코드 작성 | UI에서 어댑터·트리거 조합으로 즉시 구성 |
| 실행 결과를 로그 파일로만 확인 | 대시보드에서 실행 이력·성공/실패 통계 확인 |
| 연동 변경 시 재배포 필요 | UI에서 설정 변경 후 즉시 반영 |
| 시스템별 산발적인 연동 코드 | 단일 플랫폼에서 모든 인터페이스 통합 관리 |

---

## 핵심 기능

### 어댑터
REST, JDBC, File(CSV/JSON/XML), Kafka 네 가지 타입을 지원하며, 소스와 타겟 어댑터를 각각 등록해 조합합니다.

### 트리거
- **즉시 실행**: 웹 UI에서 버튼 클릭으로 수동 실행
- **주기 실행**: Cron 표현식으로 스케줄 설정
- **이벤트 실행**: Kafka 토픽 메시지 수신 시 자동 실행
- **파일 감지**: 지정 폴더에 파일이 생성되면 자동 실행

### 필드 매핑
소스 데이터의 필드를 타겟 구조에 맞게 변환하는 규칙을 UI에서 정의합니다.  
대소문자 변환, 문자열 연결, 날짜 포맷 변환 등 다양한 변환 함수를 지원합니다.

### 모니터링
- 기간별(1h / 6h / 24h / 7d) 실행 통계 및 처리량 차트
- 실행별 상태(SUCCESS / PARTIAL / FAILED) 및 읽기·쓰기 레코드 수
- 단계별 상세 로그 (LIFECYCLE · TRANSFORM · WRITE)

## 화면 구성

| 메뉴 | 역할 |
|------|------|
| **Dashboard** | 전체 실행 현황 요약, 처리량 차트, 최근 실행 목록 |
| **Systems** | 연계 대상 시스템 등록·관리 |
| **Adapters** | 소스·타겟 어댑터 설정 |
| **Interfaces** | 소스→타겟 연결, 매핑 규칙, 트리거 정의 |
| **Runs** | 전체 실행 이력 및 상세 로그 |
| **Settings** | 플랫폼 전역 설정 |

## 기대 효과

- 반복적인 연동 개발 공수 절감
- 비개발자도 운영 가능한 UI 기반 인터페이스 관리
- 실행 이력 중앙화로 장애 원인 추적 용이
- 새로운 시스템 추가 시 어댑터만 등록하면 즉시 연동 가능
